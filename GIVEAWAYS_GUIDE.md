# Руководство по системе конкурсов

## Обзор

Система конкурсов позволяет проводить розыгрыши игровых ключей, баллов и других призов для привлечения и удержания подписчиков канала.

## Возможности

- 🎮 Розыгрыш Steam ключей
- ⭐ Розыгрыш баллов
- 👑 Розыгрыш подписок и призов из магазина
- 🔒 Проверка подписки на канал
- ⏰ Автоматическое завершение по таймеру
- 📊 Отслеживание участников
- 🎲 Случайный выбор победителя
- 📢 Автоматическое объявление результатов

## Команды

### Для пользователей

- `/giveaway` — показать активные конкурсы
- `/mygiveaways` — мои конкурсы (в которых участвую)
- Кнопка "🎲 Участвовать" в постах канала

### Для администратора

- `/creategiveaway` — создать новый конкурс
- `/endgiveaway` — завершить конкурс досрочно

## Создание конкурса

### Формат команды

```
/creategiveaway [название] | [описание] | [тип] | [значение] | [часы]
```

### Типы призов

1. **steam_key** — Steam ключ игры
2. **points** — баллы для магазина
3. **subscription** — подписка/приз из магазина (reward_id)

### Примеры

#### Розыгрыш Steam ключа

```
/creategiveaway Portal 2 | Классическая головоломка от Valve | steam_key | XXXXX-XXXXX-XXXXX | 72
```

- Название: Portal 2
- Описание: Классическая головоломка от Valve
- Тип: steam_key
- Ключ: XXXXX-XXXXX-XXXXX
- Длительность: 72 часа (3 дня)

#### Розыгрыш баллов

```
/creategiveaway 1000 баллов | Потрати в магазине призов | points | 1000 | 48
```

- Название: 1000 баллов
- Описание: Потрати в магазине призов
- Тип: points
- Количество: 1000
- Длительность: 48 часов (2 дня)

#### Розыгрыш VIP статуса

```
/creategiveaway VIP статус | Месяц VIP привилегий | subscription | vip_badge | 72
```

- Название: VIP статус
- Описание: Месяц VIP привилегий
- Тип: subscription
- reward_id: vip_badge
- Длительность: 72 часа

## Как это работает

### 1. Создание конкурса

Админ создаёт конкурс командой `/creategiveaway`. Система:
- Создаёт запись в БД
- Публикует пост в канале с кнопкой "Участвовать"
- Запускает таймер

### 2. Участие

Пользователи нажимают кнопку "🎲 Участвовать" в посте. Система проверяет:
- ✅ Подписку на канал (если требуется)
- ✅ Возраст аккаунта (по умолчанию 7 дней)
- ✅ Уникальность участия (один раз на конкурс)

Счётчик участников обновляется в реальном времени.

### 3. Завершение

По истечении времени (или по команде `/endgiveaway`) система:
- Выбирает случайного победителя из участников
- Выдаёт приз (отправляет ключ/начисляет баллы)
- Объявляет победителя в канале
- Обновляет пост с результатами

## Настройки конкурса

### Требование подписки

По умолчанию включено. Пользователь должен быть подписан на канал для участия.

### Минимальный возраст аккаунта

По умолчанию 7 дней. Защита от мультиаккаунтов.

## Автоматизация

Система автоматически:
- Проверяет завершённые конкурсы каждые 15 минут
- Выбирает победителей
- Выдаёт призы
- Объявляет результаты

## База данных

### Таблица giveaways

Хранит информацию о конкурсах:
- giveaway_id (PK)
- title, description
- prize_type, prize_value
- start_time, end_time
- channel_post_id
- winner_user_id
- status (active/ended/cancelled)

### Таблица giveaway_participants

Хранит участников:
- giveaway_id (FK)
- user_id
- joined_at
- is_eligible

## Стратегия "Троянский конь"

Как описано в вашем запросе, эта стратегия работает так:

1. **Купите ключ** популярной игры (100-200₽)
2. **Создайте конкурс** через `/creategiveaway`
3. **Опубликуйте** в канале (автоматически)
4. **Поделитесь** ссылкой в 5-10 геймерских чатах
5. **Получите** резкий рост подписчиков

### Пример сообщения для чатов

```
Парни, тут ключ Portal 2 разыгрывают!
Подписка на канал = участие
Розыгрыш через 3 дня
[ссылка на канал]
```

### Рекомендации

- Выбирайте **хайповые** игры (новинки, популярные инди)
- Или **классику** (Portal, Bioshock, Half-Life)
- Длительность **2-3 дня** оптимальна
- После розыгрыша **часть уйдёт**, но ядро останется
- Проводите регулярно для **удержания** аудитории

## Метрики

Отслеживайте:
- Количество участников
- Прирост подписчиков за период конкурса
- Retention после конкурса (через неделю)
- Вовлечённость (голоса, клики) после конкурса

## Советы

1. **Регулярность** — проводите конкурсы раз в 1-2 недели
2. **Разнообразие** — чередуйте ключи, баллы, статусы
3. **Анонсы** — предупреждайте о конкурсах заранее
4. **Честность** — всегда выдавайте призы победителям
5. **Промо** — делитесь в тематических чатах и группах

## Безопасность

- Проверка возраста аккаунта (защита от ботов)
- Проверка подписки на канал
- Уникальность участия (один раз на конкурс)
- Атомарная выдача ключей (нет дубликатов)

## Troubleshooting

### Конкурс не завершается автоматически

Проверьте:
- Работает ли планировщик (scheduler)
- Есть ли задача `check_giveaways` в логах
- Правильно ли установлено время окончания

### Ключ не отправился победителю

Проверьте:
- Бот не заблокирован пользователем
- Ключ корректный (формат XXXXX-XXXXX-XXXXX)
- Логи на наличие ошибок отправки

### Счётчик участников не обновляется

Проверьте:
- Права бота на редактирование сообщений в канале
- channel_post_id сохранён в БД
- Callback обработчик зарегистрирован
