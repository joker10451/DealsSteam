# Система обнаружения ошибок цен (Price Glitches)

## Обзор

Бот автоматически обнаруживает аномальные скидки, которые могут быть ошибками магазинов. Такие "глитчи" публикуются с высоким приоритетом и специальными предупреждениями.

## Алгоритм обнаружения

### 1. Экстремальные скидки (95%+)

**Критическая аномалия (98%+):**
- Эмодзи: 🚨
- Заголовок: "СРОЧНО! ВОЗМОЖНАЯ ОШИБКА ЦЕНЫ!"
- Сообщение: Предупреждение о вероятной ошибке магазина
- Приоритет: Максимальный (публикуется первой)

**Высокая аномалия (95-97%):**
- Эмодзи: 🔥
- Заголовок: "ВНИМАНИЕ! АНОМАЛЬНАЯ СКИДКА!"
- Сообщение: Может быть ошибка или зачистка склада
- Приоритет: Высокий

### 2. Исторический минимум (только Steam)

Использует данные CheapShark API для сравнения с историческими минимумами.

**Критическая аномалия (цена ниже минимума на 50%+):**
- Эмодзи: ⚡️
- Заголовок: "ИСТОРИЧЕСКИЙ МИНИМУМ (АНОМАЛИЯ)!"
- Показывает процент падения относительно предыдущего минимума
- Приоритет: Максимальный

**Высокая аномалия (цена ниже минимума на 20-50%):**
- Эмодзи: 📉
- Заголовок: "НОВЫЙ ИСТОРИЧЕСКИЙ МИНИМУМ!"
- Приоритет: Высокий

### 3. Резкое падение цены

Отслеживает историю цен в собственной БД.

**Условия:**
- Цена упала более чем в 10 раз
- Скидка выросла более чем на 50%

**Или:**
- Цена упала более чем в 20 раз (по данным из Deal)

**Результат:**
- Эмодзи: ⚠️
- Предупреждение о подозрительном падении
- Приоритет: Высокий

## База данных

### Таблица `price_history`

Хранит историю цен для обнаружения резких падений:

```sql
CREATE TABLE price_history (
    id SERIAL PRIMARY KEY,
    deal_id TEXT NOT NULL,
    price NUMERIC NOT NULL,
    discount INTEGER DEFAULT 0,
    recorded_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Очистка:** Автоматически удаляются записи старше 30 дней.

### Таблица `free_game_subs`

Хранит подписчиков на уведомления о бесплатных играх:

```sql
CREATE TABLE free_game_subs (
    user_id BIGINT PRIMARY KEY,
    subscribed_at TIMESTAMPTZ DEFAULT NOW()
);
```

## Приоритет публикации

Порядок публикации в `scheduler.py`:

1. **Price Glitches** (критические и высокие аномалии)
2. **Бесплатные игры** (is_free=True)
3. **Платные игры** (сортировка по теме дня и скидке)

## Уведомления

### Подписчики на бесплатные игры

Команды:
- `/free` — подписаться на уведомления
- `/free_off` — отписаться

Уведомления отправляются автоматически при публикации бесплатной игры.

### Подписчики вишлиста

Получают уведомления о glitch'ах, если игра в их вишлисте.

### Подписчики жанров

Получают уведомления о glitch'ах в подписанных жанрах.

1. 🚨 ОШИБКА ЦЕНЫ? СРОЧНО!
2. 🔥 АНОМАЛЬНАЯ СКИДКА!

Добавляется после информации о рейтинге и историческом минимуме:

```
⚠️ ВНИМАНИЕ: Скидка 98% — вероятная ОШИБКА ЦЕНЫ!
Такие скидки почти всегда являются ошибками магазина.
Покупайте срочно — могут отменить заказ!
```

## Примеры использования

### Проверка сделки на glitch

```python
from price_glitch import check_for_glitch

glitch_info = await check_for_glitch(deal)
if glitch_info:
    print(f"Обнаружена аномалия: {glitch_info['type']}")
    print(f"Серьёзность: {glitch_info['severity']}")
    print(f"Сообщение: {glitch_info['message']}")
```

### Быстрая проверка критичности

```python
from price_glitch import is_price_glitch

if await is_price_glitch(deal):
    print("Критическая аномалия! Публикуем немедленно!")
```

### Форматирование предупреждения

```python
from price_glitch import format_glitch_alert

alert = format_glitch_alert(deal, glitch_info)
# Возвращает отформатированный HTML для Telegram
```

## Метрики

Отслеживаемые события:
- `published` — общее количество публикаций
- `publish_error` — ошибки публикации
- `wishlist_notify` — уведомления вишлиста
- `genre_notify` — уведомления по жанрам
- `free_game_notify` — уведомления о бесплатных играх

## Ограничения и рекомендации

### Rate Limiting

- CheapShark API: без жёстких лимитов, но рекомендуется кэширование (реализовано в `enricher.py`)
- Telegram API: используется `send_with_retry()` для обработки flood control

### Кэширование

- Исторические минимумы: 6 часов (TTL в `enricher.py`)
- Рейтинги Steam: 1 час

### Безопасность

- Все цены извлекаются через `_extract_price_value()` с обработкой ошибок
- Невалидные форматы цен не вызывают сбоев
- История цен автоматически очищается
