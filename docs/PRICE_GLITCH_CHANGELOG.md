# Changelog: Price Glitch Detection System

## Добавленные файлы

### `price_glitch.py`
Основной модуль системы обнаружения ошибок цен:
- `check_for_glitch(deal)` — проверяет сделку на аномалии
- `format_glitch_alert(deal, glitch_info)` — форматирует предупреждение
- `is_price_glitch(deal)` — быстрая проверка критичности
- `_extract_price_value(price_str)` — извлекает числовое значение цены

### `tests/test_price_glitch.py`
Тесты для системы обнаружения глитчей:
- Тесты экстремальных скидок (95%+, 98%+)
- Тесты подозрительных падений цены
- Тесты форматирования предупреждений
- Тесты обработки невалидных данных

### Документация
- `PRICE_GLITCH_GUIDE.md` — техническая документация для разработчиков
- `GLITCH_HUNTING_GUIDE_RU.md` — гайд для пользователей на русском

## Изменённые файлы

Добавлены таблицы и функции:

**Таблица `price_history`:**
```sql
CREATE TABLE price_history (
    id SERIAL PRIMARY KEY,
    deal_id TEXT NOT NULL,
    price NUMERIC NOT NULL,
    discount INTEGER DEFAULT 0,
    recorded_at TIMESTAMPTZ DEFAULT NOW()
```

**Функции:**
- `save_price_history(deal_id, price, discount)` — сохраняет историю цен

- `free_game_subscribe(user_id)` — подписка на бесплатные игры
- `free_game_unsubscribe(user_id)` — отписка
- `free_game_get_subscribers()` — список подписчиков

### `publisher.py`
Интеграция системы обнаружения глитчей:

**Изменения в `publish_single()`:**
- Добавлен параметр `is_priority` для приоритетных публикаций
- Проверка на glitch через `check_for_glitch(deal)`
- Приоритет заголовков: glitch > бесплатно > исторический минимум > огонь-скидка
- Добавление предупреждения об ошибке цены в пост

**Новые функции:**
- `notify_free_game_subscribers(deal)` — уведомляет подписчиков о бесплатных играх

### `scheduler.py`
Приоритизация публикаций:

**Изменения в `check_and_post()`:**
- Разделение сделок на категории: glitches, free, paid

- Уведомление подписчиков о бесплатных играх через `notify_free_game_subscribers()`

### `handlers/wishlist.py`
Добавлены команды для подписки на бесплатные игры:

**Обновлена команда `/start`:**

- Добавлена команда `/free` в список основных команд
- Добавлена команда `/genre` в дополнительные команды

## Алгоритм обнаружения

### 1. Экстремальные скидки
- **98%+** → Критическая аномалия (🚨)
- **95-97%** → Высокая аномалия (🔥)

### 2. Исторический минимум (Steam only)
- **Цена < минимум * 0.5** → Критическая аномалия (⚡️)
- **Цена < минимум * 0.8** → Высокая аномалия (📉)

### 3. Резкое падение цены
- **Падение в 10+ раз + рост скидки на 50%+** → Высокая аномалия (⚠️)
- **Падение в 20+ раз** → Высокая аномалия (⚠️)

## Приоритет публикации

1. **Price Glitches** (критические и высокие)
2. **Бесплатные игры**
3. **Платные игры** (по теме дня и скидке)

## Уведомления

### Типы уведомлений
1. **Вишлист** — при любой скидке на игру из вишлиста
2. **Жанры** — при скидке на игру подписанного жанра
3. **Бесплатные игры** — при появлении бесплатной игры (новая функция)

### Метрики
- `published` — общее количество публикаций
- `wishlist_notify` — уведомления вишлиста
- `genre_notify` — уведомления по жанрам
- `free_game_notify` — уведомления о бесплатных играх (новая метрика)

## Форматирование постов

### Заголовки (по приоритету)
1. 🚨 **ОШИБКА ЦЕНЫ? СРОЧНО!** (glitch critical)
2. 🔥 **АНОМАЛЬНАЯ СКИДКА!** (glitch high)
3. 🎁 **БЕСПЛАТНО** (free game)
4. ⚡️ **ИСТОРИЧЕСКИЙ МИНИМУМ** (historical low)
5. 🔥 **ОГОНЬ-СКИДКА** (80%+ discount)
6. Тема дня (RPG-понедельник и т.д.)

### Предупреждение
Добавляется после информации о рейтинге:

```
🚨 СРОЧНО! ВОЗМОЖНАЯ ОШИБКА ЦЕНЫ!

⚠️ ВНИМАНИЕ: Скидка 98% — вероятная ОШИБКА ЦЕНЫ!
Такие скидки почти всегда являются ошибками магазина.
Покупайте срочно — могут отменить заказ!
```

## Миграция БД

При первом запуске автоматически создаются новые таблицы:
- `price_history` — история цен
- `free_game_subs` — подписчики на бесплатные игры

Индексы создаются автоматически для оптимизации запросов.

## Производительность

### Кэширование
- Исторические минимумы: 6 часов (TTL в `enricher.py`)
- Рейтинги Steam: 1 час

### Очистка данных
- История цен: автоматическая очистка записей старше 30 дней
- Запускается при каждом вызове `cleanup_old_records()`

### Rate Limiting
- CheapShark API: без жёстких лимитов
- Telegram API: обработка через `send_with_retry()`

## Безопасность

- Все цены извлекаются с обработкой ошибок
- Невалидные форматы не вызывают сбоев
- SQL-инъекции предотвращены через параметризованные запросы
- HTML-экранирование через `esc()` для всех пользовательских данных
